# 入社時オンボーディング統合ポータル 詳細仕様書

## 1. 背景・目的

### 1.1 背景

* 入社時に対応すべき事項が多く、内容も多岐にわたる
* 現状は Notion 上の NewComers ページを起点に、各種案内を読み進める形式
* 企業理念・会社概要の理解から、IT環境構築・各種手続きまでが連続しており、以下の課題がある

  * 何をどこまで対応すればよいか分かりにくい
  * 対応漏れ・確認漏れが発生しやすい
  * 情シス・人事・財務が個別にフォローする必要がある

### 1.2 本ポータルの目的

* 入社時に必要な作業を **1ステップ1ページ形式** で整理し、迷わず進められるようにする
* Notion を正本としつつ、作業進捗を可視化するハブとして機能させる
* 入社者・人事・情シス・財務の認識を揃える

---

## 2. システムの位置づけ（全体像）

* 本ポータルは「作業を実行する場」ではなく「案内・確認・進捗管理の場」とする
* 実作業は以下の外部サービスで実施する

  * Notion
  * Slack
  * Microsoft 365 / Microsoft Defender for Business
  * Google Calendar
  * GitHub
  * freee
  * さくらインターネット（メール）

---

## 3. 利用者と権限

| ロール     | 主な利用内容                |
| ------- | --------------------- |
| 新入社員    | 各ステップを順に実施し、完了チェックを行う |
| 人事      | 契約・規程関連ステップの進捗確認      |
| 情報システム部 | IT環境・セキュリティ関連ステップの管理  |
| 財務担当    | freee登録関連の進捗確認        |
| 管理者     | ステップ構成・文言・リンク管理       |

※ 書き込み（完了操作）は本人のみ可能

---

## 4. オンボーディングの進め方（UX方針）

* ステップは **原則順番通り** に並ぶ
* 各ステップは 1ページ完結
* 各ページには以下を必ず含める

  * このステップの目的
  * 実施内容の詳細説明
  * 実作業を行う外部リンク
  * 注意点・よくあるミス
  * 「完了しました」チェック

## 6. 進捗管理機能

* 各ステップごとに以下の状態を保持

* 人事・情シス・財務は該当ステップのみ進捗確認可能
* CSV 出力対応

## 8. 非機能要件

### セキュリティ

* HTTPS 必須
* 個人情報は最小限
* 外部サービスの認証情報は保持しない
* 操作ログを保持

### 運用
* ステップ内容は管理者が柔軟に変更可能
* Notion 側の更新が即時反映される運用を前提
* バックエンドの煩雑化対処が必要
- 生年月日の入力を受け取ったら、財務担当者に通知を送り、財務担当者がfreeeの招待メールを送信（ここも自動化出来たらベスト）
- 同様に労働条件通知書および秘密保持契約書もそのステップの時点で既にメールにPDFが添付されていて、すぐに確認できるようにする
- 基本情報の入力ステップの時点で、既にメールがユーザに送られるような仕組みが良い。間に企業理念と就業規則の確認ステップを挟んで置けば、その間にユーザに担当者からメールを送ることが可能である。
- 2~3日想定らしい…
